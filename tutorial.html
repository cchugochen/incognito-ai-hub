Gemini API Setup / Gemini API 設定教學

Get a Gemini API key for Incognito PrivoAI

Updated for v28.1.6 · April 2026

English

This guide shows the fastest path to creating a Gemini API key in Google AI Studio and adding it to this extension. Exact button labels can change over time, but the overall flow stays the same: sign in, create a key, copy it once, and paste it into Options.

繁體中文

這份教學提供目前最快的流程，帶您在 Google AI Studio 建立 Gemini API 金鑰，並加入本擴充功能。 介面按鈕名稱未來可能微調，但整體步驟通常不變：登入、建立金鑰、複製一次、再貼到選項設定頁。

What you need

A Google account and access to Google AI Studio.

What you will get

A single API key you can paste into the extension's Options page.

Good to know

The key is project-level access. Keep it private and revoke unused keys.

Before you start

  • If a screen looks slightly different, keep going. Google updates AI Studio often.
  • You do not need to understand Google Cloud in depth for the basic setup.
  • If you only want local text workflows, the extension can also use a local AI model later.

開始前先知道

  • 如果畫面和截圖略有不同，通常不用緊張，Google AI Studio 介面更新很常見。
  • 完成基本設定不需要先深入理解整套 Google Cloud。
  • 如果您只想做本機文字任務，之後也可以改用本機 AI 模型。
1

Open the API key page

Go to Google AI Studio's API key page and sign in with your Google account. If Google asks you to choose a project first, that is normal.

Open Google AI Studio

開啟 API 金鑰頁面

前往 Google AI Studio 的 API 金鑰頁面，並使用您的 Google 帳號登入。 如果系統先要求您選擇專案，也屬正常流程。

2

Create a key in a project

Click the API-key creation button shown in AI Studio. For first-time users, creating the key in a new project is usually the simplest choice. If you already have a project for Gemini API use, you can place the key there instead.

在專案中建立金鑰

點擊 AI Studio 介面中的 API 金鑰建立按鈕。對第一次使用者來說，直接建立在新專案中通常最簡單。 如果您已經有專門給 Gemini API 使用的專案，也可以把金鑰建立在既有專案內。

3

Copy the key and keep it private

Copy the key shown after creation. Gemini API keys often begin with AIza. Store it carefully, avoid screenshots in public places, and do not paste it into GitHub, forums, or shared documents.

複製金鑰並妥善保管

金鑰建立完成後請立即複製。Gemini API 金鑰通常會以 AIza 開頭。 請小心保存，不要把它貼到 GitHub、論壇、共用文件，或任何可能被他人看到的地方。

4

Paste it into the extension

Open the extension's Options page, paste the key into the Gemini API Key field, and save. If you are unsure which model to choose, start with the default or recommended Flash/Lite option shown in the UI. Model names can change, so the safest rule is to begin with the default and adjust later if needed.

Once saved, Gemini-based features are ready to use.

貼到擴充功能選項頁

打開擴充功能的「選項」頁面，把金鑰貼到「Gemini API 金鑰」欄位後儲存。 如果您不確定該選哪個模型，先使用介面中預設或建議的 Flash / Lite 選項即可。 模型名稱未來可能變動，因此最穩妥的做法是先用預設值，之後再視需求調整。

儲存完成後，就可以使用基於 Gemini 的功能了。

Key safety tips

  • Use a dedicated key for this extension instead of reusing a key from another project.
  • Delete or rotate keys you no longer need.
  • If you manage keys in Google Cloud, restrictions are a good idea, but test carefully because browser-extension clients may behave differently from server apps.

金鑰安全建議

  • 建議為本擴充功能使用專用金鑰，不要和其他專案共用。
  • 不再使用的金鑰，請主動刪除或輪替。
  • 若您在 Google Cloud 管理金鑰，設定限制通常是好習慣；但瀏覽器擴充功能與伺服器型應用不同，套用限制後請務必實際測試。

Optional: local AI for text tasks

This project can also connect to Ollama, LM Studio, or another OpenAI-compatible local endpoint for text-only tasks. That is useful when you want private local processing for chat or translation. Voice, OCR, image, and PDF features still rely on Gemini.

選配：文字任務可改用本機 AI

本專案也支援連線到 Ollama、LM Studio 或其他 OpenAI 相容的本機端點，處理純文字任務。 如果您想讓聊天或翻譯走本機處理，這會很實用；但語音、OCR、圖片與 PDF 功能仍然需要 Gemini。

References: Gemini API Quickstart, Google Cloud API Keys documentation.
