Incognito PrivoAI Privacy Policy

Last updated: April 2026 · Applies to project version v28.1.6

Incognito PrivoAI is built so AI requests go directly from your browser to the AI service you choose. We do not run an application backend, we do not maintain user accounts, and we do not collect or sell your content.

Incognito PrivoAI 的設計原則，是讓 AI 請求直接由您的瀏覽器送往您選擇的 AI 服務。 我們不營運應用程式後端、不建立使用者帳號，也不收集或販售您的內容資料。

No Relay Server

Requests go from your browser to Google Gemini or your own local endpoint.

Settings Stay In Browser

Preferences and API key settings are stored in Chrome extension storage on your device profile.

Temporary Working Data

Some features briefly stage text or file data locally only long enough to complete the workflow.

Local Model Option

If you enable a local model, requests go to the local server you configure instead of our infrastructure.

1. What the extension stores

What is stored, where it is stored, and why.

Settings and API key

The extension stores your Gemini API key, preferred languages, preset prompts, AI mode, and local-model settings in chrome.storage.sync. Chrome may sync this data across signed-in browsers if you enable Chrome Sync. According to Chrome's storage documentation, storage.local and storage.sync are not intended for confidential data, so use a dedicated or revocable API key if that matters to your workflow.

擴充功能會將 Gemini API 金鑰、偏好語言、預設提示語、AI 模式與本機模型設定儲存在 chrome.storage.sync。若您啟用 Chrome Sync，這些設定可能會同步到您登入的其他 Chrome 瀏覽器。 依 Chrome 官方文件，storage.local 與 storage.sync 不應視為機密資料保險箱； 若您在意這點，建議使用可撤銷、可輪替的專用 API 金鑰。

Temporary content used during a task

Some workflows temporarily place text or task state in chrome.storage.local so one extension page can hand work to another page or to the background worker. This temporary data is meant for task completion and is removed after handoff or use.

某些流程會暫時把文字內容或任務狀態放在 chrome.storage.local，讓不同擴充頁面或背景 worker 可以完成交接。 這些資料的用途是完成當前流程，通常會在交接完成或使用後刪除。

Incognito chat history

The chat history in the Incognito chat pages is designed to live only in page memory while that tab is open. Closing the tab clears that in-memory history.

無痕對話頁面的聊天紀錄，設計上只會在該分頁開啟期間保留於頁面記憶體中。 關閉分頁後，這段記憶體中的歷史內容就會一併消失。

2. Where your data is sent

The extension does not proxy your requests through a developer-run service.

English

  • When you use Gemini features, requests are sent directly from your browser to Google's Gemini API endpoint.
  • When you use a local model, requests are sent directly to the local or LAN endpoint that you configured.
  • The project does not operate its own AI relay server for your prompts, files, or transcripts.
  • Google's handling of data on its side is governed by Google's own policies and terms.

繁體中文

  • 使用 Gemini 功能時，請求會直接從您的瀏覽器送往 Google 的 Gemini API 端點。
  • 使用本機模型時，請求會直接送往您自行設定的本機或區網端點。
  • 本專案不營運自己的 AI 中繼伺服器來轉送您的提示、檔案或語音內容。
  • 資料進入 Google 端後的處理方式，仍以 Google 自己的政策與條款為準。

3. Permissions used by the extension

These permissions match the current manifest and feature set in v28.1.6.

activeTab

Used to work with the current tab when you trigger webpage processing.

用於在您啟動網頁處理時，操作目前使用中的分頁。

scripting

Used to read page content and run the webpage translation or extraction flow.

用於讀取頁面內容，以及執行網頁翻譯或擷取流程。

debugger

Used only for the current-page translation workflow that captures a full-page screenshot, then detaches.

僅用於「翻譯目前網頁」流程中的全頁截圖，完成後即解除附加。

storage

Used to store settings, presets, language preferences, and the API key you choose to keep in the extension.

用於儲存設定、預設提示語、語言偏好，以及您選擇保存在擴充功能中的 API 金鑰。

Host access

Broad host access is required so the extension can process arbitrary webpages and connect to user-configured local AI endpoints.

因為擴充功能需要處理任意網頁，並連線到使用者自行設定的本機 AI 端點，所以需要較廣的主機存取範圍。

4. Practical privacy guidance

Good defaults you can choose based on your own risk tolerance.

  • Use a dedicated Gemini API key for this extension instead of reusing a key from other projects.
  • Rotate or revoke keys you no longer need.
  • If you do not want settings synced across browsers, turn off Chrome Sync for the profile that stores the key.
  • For sensitive text-only workflows, consider using a local model endpoint.
  • 建議為本擴充功能使用專用的 Gemini API 金鑰，不要與其他專案共用。
  • 不再使用的金鑰，請主動輪替或撤銷。
  • 如果您不希望設定同步到其他瀏覽器，可關閉儲存該金鑰之 Chrome 個人資料的同步功能。
  • 若是高敏感、且僅限文字的工作流程，可考慮使用本機模型端點。
